**Note: KSampler settings**
You can test and find the best setting by yourself. The following table is for reference.
| Parameters   | Qwen Team | Comfy Original | with 4steps LoRA |
|--------|---------|------------|---------------------------|
| Steps  | 50      | 20         | 4                         |
| CFG    | 4.0     | 2.5        | 1.0                       |

**Note: Raw latent version**
Since the **TextEncodeQwenImageEditPlus** node will scale the image to 1M pixels, you might find that the output has a pixel-shifting issue.

The **Raw latent version** is to provide the raw latent to the sampler. This is the way you can provide the raw latent as an input condition.

## Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `width` | int | `1024` | Output width in pixels |
| `height` | int | `1024` | Output height in pixels |
| `seed` | int | `365832727981113` | Random seed (use -1 for random) |
| `steps` | int | `4` | Number of sampling steps |
| `cfg` | float | `1` | CFG scale (guidance strength) |
| `sampler_name` | enum | `euler` | Sampler algorithm |
| `scheduler` | enum | `simple` | Noise scheduler |
| `filename_prefix` | string | `ComfyUI` | Output filename prefix |

## Usage

```bash
c3 comfyui run image_qwen_image_edit_2509 \
  --prompt "your prompt here" \
  --output my_output \
  --width 1024 \
  --height 1024 \
  --steps 4 \
  --cfg 1
```
