## Overview

There are two HyperClaw-related namespaces in the TS SDK:

- `client.agent`
- `client.deployments`

`client.agent` covers account metadata.
`client.deployments` covers live runtime containers.

## URL Model

The TS SDK splits the product API and agents API cleanly:

- `apiUrl`
  - main HyperCLI REST API
- `agentsApiBaseUrl`
  - HyperClaw deployments REST API
- `agentsWsUrl`
  - HyperClaw backend WS base for shell/logs
- `client.agent.baseUrl`
  - inference base derived from `agentsApiBaseUrl` by replacing `/api` with `/v1`

Recommended defaults:

```bash
HYPERCLI_API_URL=https://api.hypercli.com
AGENTS_API_BASE_URL=https://api.hypercli.com/agents
AGENTS_WS_URL=wss://api.agents.hypercli.com/ws
```

## Create

```ts
const agent = await client.deployments.create({
  name: "docs-demo",
  size: "medium",
  start: true,
  env: { APP_ENV: "prod" },
});
```

This generic path does not inject any routes. Use `createOpenClaw(...)` when you want the standard root gateway and `desktop` routes.

Create/start options:

- `name`
- `size`
- `cpu`
- `memory`
- `config`
- `env`
- `ports`
- `routes`
- `command`
- `entrypoint`

- `gatewayToken`
- `dryRun`

## OpenClaw Presets

Use the OpenClaw-specific helpers when you want the SDK to provide the standard gateway and desktop routes:

```ts
const agent = await client.deployments.createOpenClaw({
  name: "docs-demo",
  start: true,
});
```

Default OpenClaw routes are:

- `openclaw`
  - `prefix: ""`
  - `port: 18789`
- `desktop`
  - `prefix: "desktop"`
  - `port: 3000`

Route semantics are explicit:

- omit `routes`
  - SDK injects the standard OpenClaw routes
- pass `routes: {}`
  - no routes are created
- pass a non-empty `routes` object
  - SDK uses exactly what you passed

The same contract applies to `startOpenClaw(...)`.

## Routes

Routes are a generic dict keyed by route name. Outside the OpenClaw helper methods, the SDK does not inject route defaults.

```ts
const agent = await client.deployments.create({
  name: "docs-demo",
  start: true,
  routes: {
    app: { prefix: "app", port: 8080, auth: false },
    admin: { prefix: "admin", port: 8081, auth: true },
  },
});
```

## Files And Exec

```ts
const result = await agent.exec("uname -a", { timeout: 30 });
console.log(result.stdout, result.exitCode);

await agent.fileWrite("/workspace/notes.txt", "hello");
const text = await agent.fileRead("/workspace/notes.txt");

await agent.cpTo("./local.txt", "/workspace/local.txt");
await agent.cpFrom("/workspace/output.log", "./output.log");
```

## Logs And Shell

Recommended helpers:

```ts
const logs = await client.deployments.logsConnect(agent.id, { tailLines: 200 });
const shell = await client.deployments.shellConnect(agent.id, "/bin/bash");
```

These use:

- `/api/deployments/{id}/logs/token`
- `/api/deployments/{id}/shell/token`
- backend `/ws/logs/{agent_id}`
- backend `/ws/shell/{agent_id}`

Prefer these over direct executor-era HTTP assumptions.

### Gateway URL Semantics

If the backend does not explicitly send `openclaw_url` / `gateway_url`, the SDK falls back to:

```ts
`wss://${agent.hostname}`
```

That matches the current root-host gateway model.
