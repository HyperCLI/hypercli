## Overview

There are two HyperClaw-related namespaces in the TS SDK:

- `client.agent`
- `client.deployments`

`client.agent` covers account metadata.
`client.deployments` covers live runtime containers.

## URL Model

The TS SDK splits the product API and agents API cleanly:

- `apiUrl`
  - main HyperCLI REST API
- `agentsApiBaseUrl`
  - HyperClaw deployments REST API
- `agentsWsUrl`
  - HyperClaw backend WS base for shell/logs
- `client.agent.baseUrl`
  - inference base derived from `agentsApiBaseUrl` by replacing `/api` with `/v1`

Recommended defaults:

```bash
HYPER_API_BASE=https://api.hypercli.com
AGENTS_API_BASE_URL=https://api.hypercli.com/agents
AGENTS_WS_URL=wss://api.agents.hypercli.com/ws
```

## `client.agent`

Use this namespace for:

- `plans()`
- `subscriptions()`
- `subscriptionSummary()`
- `entitlements()`
- `entitlementInstances()`
- `updateSubscription(subscriptionId, { bundle })`
- `cancelSubscription(subscriptionId)`
- `models()`
- `discoveryHealth()`
- `discoveryConfig()`

The key split is:

- `entitlements()`
  effective current account state, including pooled inference, slot inventory, active entitlement count, and `billingResetAt`
- `subscriptions()`
  recurring card-backed billing objects such as Stripe subscriptions, including `cancelAtPeriodEnd`
- `entitlementInstances()`
  concrete 1:1 entitlement grants with expiry, tags, features, and related agent bindings
- `subscriptionSummary()`
  combined account view that includes both the effective `entitlements` object and the active subscription list
- `updateSubscription()`
  unified recurring billing mutation surface for upgrade, downgrade, and cancel-at-period-end

Example:

```ts
const summary = await client.agent.subscriptionSummary();
const entitlements = await client.agent.entitlements();
const entitlementItems = await client.agent.entitlementInstances();
const subscriptions = await client.agent.subscriptions();

await client.agent.updateSubscription("sub_123", { bundle: { large: 1 } });
```

Use an empty bundle to cancel at period end:

```ts
await client.agent.updateSubscription("sub_123", { bundle: {} });
```

## `client.deployments`

Use this namespace for:

- create / start / stop / delete
- slot inventory / metrics
- files
- exec
- backend WS logs and shells
- OpenClaw gateway access

`budget()` now returns:

- `slots`
  exact-tier granted / used / available counts
- `pooled_tpd`
  additive inference across active entitlements
- `size_presets`
  runtime CPU / memory shapes for each tier

For account-facing billing and capacity screens, prefer `client.agent.entitlements()` or `client.agent.subscriptionSummary()` over `budget()`. `budget()` is runtime deployment capacity data, not recurring billing state.

## Create

```ts
const agent = await client.deployments.create({
  name: "docs-demo",
  size: "medium",
  start: true,
  env: { APP_ENV: "prod" },
});
```

This generic path does not inject any routes. Use `createOpenClaw(...)` when you want the standard root gateway and `desktop` routes.

Create/start options:

- `name`
- `size`
- `cpu`
- `memory`
- `config`
- `tags`
- `env`
- `ports`
- `routes`
- `command`
- `entrypoint`
- `image`
- `registryUrl`
- `registryAuth`
- `gatewayToken`
- `dryRun`

`registryUrl`, `registryAuth`, `syncRoot`, and `syncEnabled` are generic launch options. They belong to the base deployment contract, not to OpenClaw specifically.

## OpenClaw Presets

Use the OpenClaw-specific helpers when you want the SDK to provide the standard gateway and desktop routes:

```ts
const agent = await client.deployments.createOpenClaw({
  name: "docs-demo",
  start: true,
  tags: ["team=dev"],
});
```

Default OpenClaw routes are:

- `openclaw`
  - `prefix: ""`
  - `port: 18789`
- `desktop`
  - `prefix: "desktop"`
  - `port: 3000`

Route semantics are explicit:

- omit `routes`
  - SDK injects the standard OpenClaw routes
- pass `routes: {}`
  - no routes are created
- pass a non-empty `routes` object
  - SDK uses exactly what you passed

The same contract applies to `startOpenClaw(...)`.

OpenClaw helpers only add defaults on top of that generic launch surface:

- `image`
  defaults to the published OpenClaw image when omitted
- `routes`
  defaults to the standard `openclaw` and `desktop` routes when omitted
- `env.HOME`
  defaults to `/app`
- `syncRoot`
  defaults to `/app`
- `syncEnabled`
  defaults to `true`

Registry settings are still inherited from the generic deployment API:

```ts
const agent = await client.deployments.createOpenClaw({
  name: "private-image-demo",
  registryUrl: "git.nedos.co",
  registryAuth: { username: "ci", password: "token" },
});
```

## Routes

Routes are a generic dict keyed by route name. Outside the OpenClaw helper methods, the SDK does not inject route defaults.

```ts
const agent = await client.deployments.create({
  name: "docs-demo",
  start: true,
  routes: {
    app: { prefix: "app", port: 8080, auth: false },
    admin: { prefix: "admin", port: 8081, auth: true },
  },
});
```

## Files And Exec

```ts
const result = await agent.exec("uname -a", { timeout: 30 });
console.log(result.stdout, result.exitCode);

await agent.fileWrite("/workspace/notes.txt", "hello");
const text = await agent.fileRead("/workspace/notes.txt");

await agent.cpTo("./local.txt", "/workspace/local.txt");
await agent.cpFrom("/workspace/output.log", "./output.log");
```

## Logs And Shell

Recommended helpers:

```ts
const logs = await client.deployments.logsConnect(agent.id, { tailLines: 200 });
const shell = await client.deployments.shellConnect(agent.id, "/bin/bash");
```

These use:

- `/api/deployments/{id}/logs/token`
- `/api/deployments/{id}/shell/token`
- backend `/ws/logs/{agent_id}`
- backend `/ws/shell/{agent_id}`

Prefer these over direct executor-era HTTP assumptions.

To mint a child key for one exact agent:

```ts
const scoped = await client.deployments.createScopedKey(agent.id, "agent-client");
console.log(scoped.api_key);
```

### Gateway URL Semantics

If the backend does not explicitly send `openclaw_url` / `gateway_url`, the SDK falls back to:

```ts
`wss://${agent.hostname}`
```

That matches the current root-host gateway model.
