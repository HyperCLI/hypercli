
- `jobs`
- `instances`
- `renders`
- `files`
- `keys`
- `user`
- `agent`
- `deployments`

## Errors

The HTTP layer throws `APIError` for non-success responses.

```ts
try {
  await client.jobs.get("missing-id");
} catch (error) {
  if (error instanceof APIError) {
    console.error(error.statusCode, error.detail);
  }
}
```

## Logs Helpers

```ts
import { LogStream, streamLogs, fetchLogs } from "@hypercli.com/sdk";
```

- `LogStream`: stateful WebSocket log client
- `streamLogs()`: convenience helper that pushes log lines to a callback
- `fetchLogs()`: fetch a job's existing logs as lines
