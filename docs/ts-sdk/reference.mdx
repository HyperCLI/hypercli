## Main Client

```ts
import { HyperCLI } from "@hypercli.com/sdk";

const client = new HyperCLI({
  apiKey: "hyper_...",
  apiUrl: "https://api.hypercli.com",
  agentApiKey: "sk-...",
  agentsApiBaseUrl: "https://api.hypercli.com/agents",
  agentsWsUrl: "wss://api.agents.hypercli.com/ws",
  timeout: 30_000,
});
```

## `agent`

Major methods:

- `plans()`
- `currentPlan()`
- `subscriptions()`
- `subscriptionSummary()`
- `entitlements()`
- `entitlementInstances()`
- `purchaseEntitlementFromBalance(planId, { duration, tags? })`
- `redeemGrantCode(code)`
- `updateSubscription(subscriptionId, { bundle })`
- `cancelSubscription(subscriptionId)`
- `models()`
- `discoveryHealth()`
- `discoveryConfig()`

Important account types:

- `HyperAgentSubscription`
  recurring billing object with `cancelAtPeriodEnd`, `currentPeriodEnd`, `planAgentTier`, and `slotGrants`
- `HyperAgentEntitlements`
  effective account state with `slotInventory`, `pooledTpd`, `activeEntitlementCount`, and `billingResetAt`
- `HyperAgentEntitlement`
  concrete 1:1 entitlement instance, including expiry, tags, features, and related agent bindings
- `HyperAgentSubscriptionSummary`
  combined view with `entitlements`, `subscriptions`, `activeSubscriptions`, and `billingResetAt`
- `HyperAgentSubscriptionMutationResult`
  message wrapper returned by recurring subscription updates
- `HyperAgentGrant`
  unapplied or applied grant record with `type`, `planId`, `duration`, `code`, `tags`, and `appliedAt`
- `HyperAgentGrantRedemptionResponse`
  wrapper returned by balance purchases and grant-code redemption with both the `grant` and resulting `entitlement`
