## Install

```bash
npm install @hypercli.com/sdk
```

Requirements:

- Node.js 18+
- ESM-compatible runtime

## What The TS SDK Covers

The SDK combines:

- HyperCLI platform REST APIs
- HyperClaw deployment lifecycle APIs
- backend WebSocket helpers for logs and shells
- OpenClaw gateway client and convenience wrappers

```ts
import { HyperCLI } from "@hypercli.com/sdk";

const client = new HyperCLI({
  apiKey: process.env.HYPER_API_KEY,
  agentApiKey: process.env.HYPER_AGENTS_API_KEY,
});
```

## Namespaces

Platform:

- `billing`
- `jobs`
- `instances`
- `renders`
- `files`
- `keys`
- `user`

HyperClaw:

- `agent`
- `deployments`

## Current Route Model

The SDK assumes the current OpenClaw route defaults:

- root host = gateway / agent
- `desktop-<host>` = desktop
- custom routes come from the `routes` dict

That means:

- `agent.publicUrl` is the root host
- `agent.desktopUrl` / `agent.vncUrl` point to `desktop-<host>`
- `agent.shellUrl` is only populated when you explicitly define a `shell` route

## Recommended Access Patterns

Preferred current helpers:

- `deployments.logsConnect()`
- `deployments.shellConnect()`
- `OpenClawAgent.gateway()`
- `OpenClawAgent.waitReady()`

Use the backend WebSocket helpers for shells/logs and the gateway client for OpenClaw behavior.

Legacy direct executor helpers still exist in the SDK, but they are not the default OpenClaw path anymore.

## Quick Start

```ts
import { HyperCLI } from "@hypercli.com/sdk";

const client = new HyperCLI({
  apiKey: "hyper_...",
  agentApiKey: "sk-...",
});

const agent = await client.deployments.createOpenClaw({
  name: "docs-demo",
  start: true,
});

const running = await client.deployments.waitRunning(agent.id);
console.log(running.publicUrl);
console.log(running.desktopUrl);
```
