## Overview

The Python SDK exposes two HyperClaw-related namespaces:

- `client.agent`
  - plans
  - model metadata
  - provider config snippets
- `client.deployments`
  - agent lifecycle
  - files
  - exec
  - shell and logs WebSockets
  - OpenClaw gateway access

```python
from hypercli import HyperCLI

client = HyperCLI(
    api_key="hyper_...",
    agent_api_key="sk-...",
)
```

## `client.agent`

Use `client.agent` for account-level metadata, billing-backed subscriptions, effective entitlements, and configuration snippets:

```python
plans = client.agent.plans()
summary = client.agent.subscription_summary()
entitlements = client.agent.entitlements()
entitlement_items = client.agent.entitlement_instances()
subscriptions = client.agent.subscriptions()
models = client.agent.models()
health = client.agent.discovery_health()
config = client.agent.discovery_config()
```

The key split is:

- `entitlements()`
  effective current account state, including pooled inference, slot inventory, active entitlement count, and `billing_reset_at`
- `subscriptions()`
  recurring card-backed billing objects such as Stripe subscriptions, including `cancel_at_period_end`
- `entitlement_instances()`
  concrete 1:1 entitlement grants with expiry, tags, features, and related agent bindings
- `subscription_summary()`
  combined account view that includes both the effective `entitlements` object and the active subscription list
- `update_subscription()`
  unified recurring billing mutation surface for upgrade, downgrade, and cancel-at-period-end

Update a recurring billing subscription:

```python
client.agent.update_subscription("sub_123", {"large": 1})
```

Use an empty bundle to cancel at period end:

```python
client.agent.update_subscription("sub_123", {})
```

This namespace also exposes convenience helpers such as:

- `chat()`
- `openai()`

Those are account/inference helpers, not agent runtime helpers.

## URL Model

The Python SDK uses separate URLs for the primary product API and the agents API:

- `client.api_url`
  - main HyperCLI API
- `client.deployments`
  - uses `AGENTS_API_BASE_URL`
- deployment shell/log streams
  - use `AGENTS_WS_URL`
- `client.agent`
  - uses the agents host for inference, deriving `/v1` from `AGENTS_API_BASE_URL`

Recommended defaults:

```bash
HYPER_API_BASE=https://api.hypercli.com
AGENTS_API_BASE_URL=https://api.hypercli.com/agents
AGENTS_WS_URL=wss://api.agents.hypercli.com/ws
```

## `client.deployments`

This namespace manages actual agent pods.

### Create

```python
agent = client.deployments.create(
    name="docs-demo",
    size="medium",
    start=True,
    env={"APP_ENV": "prod"},
)
```

This generic path does not inject any routes. Use `create_openclaw(...)` when you want the standard root gateway and `desktop` routes.

Accepted create/start options:

- `name`
- `size`
- `config`
- `tags`
- `env`
- `ports`
- `routes`
- `command`
- `entrypoint`
- `image`
- `registry_url`
- `registry_auth`
- `gateway_token`
- `dry_run`
- `start`

`registry_url`, `registry_auth`, `sync_root`, and `sync_enabled` are generic launch options. They belong to the base deployment contract, not to OpenClaw specifically.

### OpenClaw Presets

Use the OpenClaw helpers when you want the SDK to supply the standard OpenClaw routes:

```python
agent = client.deployments.create_openclaw(
    name="docs-demo",
    start=True,
    heartbeat={"every": "0m"},  # disable upstream OpenClaw heartbeat runs
    tags=["team=dev"],
)
```

Default OpenClaw routes are:

- `openclaw`
  - `prefix=""`
  - `port=18789`
- `desktop`
  - `prefix="desktop"`
  - `port=3000`

Route semantics are explicit:

- omit `routes`
  - SDK injects the standard OpenClaw routes
- pass `routes={}`
  - no routes are created
- pass a non-empty `routes` dict
  - SDK uses exactly what you passed

The same contract applies to `start_openclaw(...)`.

OpenClaw helpers only add defaults on top of that generic launch surface:

- `image`
  defaults to the published OpenClaw image when omitted
- `routes`
  defaults to the standard `openclaw` and `desktop` routes when omitted
- `sync_root`
  defaults to `/home/node`
- `sync_enabled`
  defaults to `True`
- `heartbeat`
  maps to upstream OpenClaw config at `config.agents.defaults.heartbeat`

Omit `heartbeat` to keep upstream defaults. Pass values like `heartbeat={"every": "0m"}` to disable heartbeat runs, or `heartbeat={"every": "1h", "target": "last"}` to tune them.

Registry fields are still inherited from the generic deployment API:

```python
agent = client.deployments.create_openclaw(
    name="private-image-demo",
    registry_url="git.nedos.co",
    registry_auth={"username": "ci", "password": "token"},
)
```

### Route Configuration

Routes are passed as a plain dict. Outside the OpenClaw helper methods, the SDK does not inject route defaults.

Example:

```python
agent = client.deployments.create(
    name="docs-demo",
    start=True,
    routes={
        "app": {"prefix": "app", "port": 8080, "auth": False},
        "admin": {"prefix": "admin", "port": 8081, "auth": True},
    },
)
```

## Lifecycle

```python
agents = client.deployments.list()
current = client.deployments.get(agent.id)

agent = client.deployments.start(agent.id)
agent = client.deployments.stop(agent.id)
client.deployments.delete(agent.id)

budget = client.deployments.budget()
metrics = client.deployments.metrics(agent.id)
```

`budget` now reports slot inventory plus pooled inference, rather than a summed CPU / memory billing pool. Expect:

- `slots`
  exact-tier granted / used / available counts
- `pooled_tpd`
  inference pooled across all active entitlements
- `size_presets`
  runtime shapes for `small`, `medium`, and `large`

For account-facing billing and capacity screens, prefer `client.agent.entitlements()` or `client.agent.subscription_summary()` over `budget()`. `budget()` is runtime deployment capacity data, not recurring billing state.

Wait for a running pod:

```python
agent = client.deployments.wait_running(agent.id, timeout=300, poll_interval=5)
```

Or on the hydrated object:

```python
agent.wait_running(timeout=300)
```

## `Agent` Object

A hydrated `Agent` contains:

- `id`
- `user_id`
- `pod_id`
- `pod_name`
- `state`
- `name`
- `hostname`
- `tags`
- `jwt_token`
- `jwt_expires_at`
- `routes`
- `command`
- `entrypoint`
- `ports`

Important computed properties:

- `public_url`
  - root host
- `desktop_url`
  - `desktop-<host>`
- `vnc_url`
  - alias of `desktop_url`
- `shell_url`
  - only if an explicit `shell` route exists
- `executor_url`
  - alias of `shell_url`
- `route_url(route_name, default_prefix=None)`
- `route_requires_auth(route_name)`
- `is_running`

## Exec, Files, Copy

```python
result = agent.exec("uname -a", timeout=30)
print(result.stdout, result.stderr, result.exit_code)

agent.file_write("/workspace/notes.txt", "hello")
print(agent.file_read("/workspace/notes.txt"))

agent.cp_to("./local.txt", "/workspace/local.txt")
agent.cp_from("/workspace/output.log", "./output.log")
```

Equivalent deployment helpers:

- `client.deployments.exec(...)`
- `files_list()`
- `file_read()`
- `file_write()`
- `file_delete()`
- `cp_to()`
- `cp_from()`

## Logs And Shell

Recommended current helpers:

```python
async for line in agent.logs_stream_ws(tail_lines=200):
    print(line)

ws = await agent.shell_connect(shell="/bin/bash")
```

These use:

- `/api/deployments/{id}/logs/token`
- `/api/deployments/{id}/shell/token`
- backend `/ws/logs/{agent_id}`
- backend `/ws/shell/{agent_id}`

Prefer these over legacy direct executor-style methods.

## `OpenClawAgent`

When the backend includes gateway metadata, the SDK hydrates `OpenClawAgent`.

`OpenClawAgent` adds:

- `gateway_url`
- `gateway_token`
- `resolve_gateway_token()`
- `gateway()`
- `connect()`
- `gateway_status()`
- `wait_ready()`
- `config_get()`
- `config_schema()`
- `config_patch()`
- `config_apply()`
- `models_list()`
- `sessions_list()`
- `channels_status()`
- `channels_logout()`
- `web_login_start()`
- `web_login_wait()`
- `workspace_files()`
- `file_get()`
- `file_set()`
- `chat_history()`
- `chat_send_message()`
- `chat_send()`
- `provider_upsert()`
- `provider_remove()`
- `model_upsert()`
- `model_remove()`
- `set_default_model()`
- `set_memory_search()`
- `channel_upsert()`
- `channel_patch()`
- `telegram_upsert()`
- `slack_upsert()`
- `discord_upsert()`
- `cron_list()`

`resolve_gateway_token()` now uses `GET /api/deployments/{id}/inference/token`. Use that endpoint for scoped agent keys instead of reading `OPENCLAW_GATEWAY_TOKEN` from `/env`.

To mint a child key for one exact agent:

```python
scoped = client.deployments.create_scoped_key(agent.id, name="agent-client")
print(scoped["api_key"])
```

### Gateway URL Semantics

The SDK now treats the root host as the default gateway host.

If the backend does not explicitly return `openclaw_url` / `gateway_url`, `OpenClawAgent` falls back to:

```python
f"wss://{agent.hostname}"
```

## Gateway Example

```python
agent = client.deployments.get("agent-id")
assert hasattr(agent, "gateway")

gw = agent.gateway()

async def main():
    async with gw:
        await gw.wait_ready(timeout=300)
        cfg = await gw.config_get()
        print(cfg)
```

See [Gateway Client SDK](/sdk/gateway) for the low-level RPC surface.
