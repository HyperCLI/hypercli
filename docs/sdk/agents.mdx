## Overview

The Python SDK exposes two separate namespaces:

- `client.agent`: HyperClaw subscription, model metadata, and convenience chat helpers
- `client.deployments`: OpenClaw and generic agent runtime lifecycle, files, exec, streaming, and gateway access

```python
from hypercli import HyperCLI

client = HyperCLI(api_key="...", agent_api_key="sk-...")
```

## HyperClaw Metadata

```python
plans = client.agent.plans()
models = client.agent.models()
key = client.agent.key_status()
health = client.agent.discovery_health()
config = client.agent.discovery_config()
```

`client.agent` also exposes:

- `openai()`
- `chat()`

## Deployments Lifecycle

```python
agent = client.deployments.create(
    name="docs-demo",
    size="medium",
    env={"APP_ENV": "prod"},
    start=True,
)

agents = client.deployments.list()
current = client.deployments.get(agent.id)
client.deployments.stop(agent.id)
client.deployments.start(agent.id)
client.deployments.delete(agent.id)
```

Create and start options include:

- `name`
- `size`
- `cpu`
- `memory`
- `config`
- `env`
- `ports`
- `routes`
- `command`
- `entrypoint`
- `image`
- `registry_url`
- `registry_auth`
- `gateway_token`
- `dry_run`

## Budget And Metrics

```python
budget = client.deployments.budget()
metrics = client.deployments.metrics(agent.id)
```

## Exec, Files, Logs, And Shells

These operations are available on both `client.deployments` and hydrated `Agent` instances.

```python
result = client.deployments.exec(agent, "ls -la /workspace", timeout=30)
print(result.stdout, result.exit_code)

client.deployments.file_write(agent, "/workspace/notes.txt", "hello")
print(client.deployments.file_read(agent, "/workspace/notes.txt"))
client.deployments.cp_to(agent, "./local.txt", "/workspace/local.txt")
client.deployments.cp_from(agent, "/workspace/output.log", "./output.log")
```

Streaming helpers:

- `logs_stream()`
- `logs_stream_ws()`
- `shell_connect()`

- `sessions_list()`
- `cron_list()`
- `chat_history()`
- `chat_send_message()`
- `chat_send()`

## Data Objects

### Agent

`Agent` includes the backend identity plus runtime metadata such as:

- `id`
- `pod_id`
- `pod_name`
- `state`
- `hostname`
- `cpu`
- `memory`
- `jwt_token`
- `routes`
- `command`
- `entrypoint`
- `ports`

Computed properties:

- `public_url`
- `vnc_url`
- `shell_url`
- `executor_url`
- `is_running`
