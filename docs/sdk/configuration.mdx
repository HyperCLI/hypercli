## Resolution Order

The Python SDK resolves config in this order:

1. Explicit constructor arguments
2. Environment variables
3. `~/.hypercli/config`
4. Built-in defaults

## Environment Variables

| Variable | Description |
| --- | --- |
| `HYPERCLI_API_KEY` | Main HyperCLI API key |
| `HYPERCLI_API_URL` | REST API base URL. Defaults to `https://api.hypercli.com` |
| `HYPERCLI_WS_URL` | Optional override for log and shell WebSocket base URLs |
| `AGENTS_API_BASE_URL` | HyperClaw deployments API base. Defaults to `https://api.hypercli.com/agents` |
| `AGENTS_WS_URL` | HyperClaw backend WebSocket base for logs and shell. Defaults to `wss://api.agents.hypercli.com/ws` |

## URL Model

The Python SDK now treats the main API and the agents API as separate surfaces:

- `HYPERCLI_API_URL`
  - primary product/backend API
  - billing
  - jobs
  - keys
  - user/account APIs
- `HYPERCLI_WS_URL`
  - primary WebSocket base for non-agent flows that derive from the main API
- `AGENTS_API_BASE_URL`
  - HyperClaw deployments REST base
  - agent lifecycle
  - files
  - exec
  - desktop tokens
  - deployment metadata
- `AGENTS_WS_URL`
  - HyperClaw backend WS base
  - `/logs/{agent_id}`
  - `/shell/{agent_id}`

Default production values:

```bash
HYPERCLI_API_URL=https://api.hypercli.com
HYPERCLI_WS_URL=wss://api.hypercli.com
AGENTS_API_BASE_URL=https://api.hypercli.com/agents
AGENTS_WS_URL=wss://api.agents.hypercli.com/ws
```

Default dev values:

```bash
HYPERCLI_API_URL=https://api.dev.hypercli.com
HYPERCLI_WS_URL=wss://api.dev.hypercli.com
AGENTS_API_BASE_URL=https://api.dev.hypercli.com/agents
AGENTS_WS_URL=wss://api.agents.dev.hypercli.com/ws
```

### OpenClaw / Inference Base URLs

The OpenClaw config generated by `hyper agent config openclaw` uses the agents host as the inference base.

Important rule:

- `anthropic-messages` providers use the base host without `/v1`
- OpenClaw appends `/v1/messages` itself
- OpenAI-compatible embeddings and completions use `/v1`

So the generated provider config should look like:

```json
{
  "models": {
    "providers": {
      "kimi-coding": {
        "baseUrl": "https://api.agents.hypercli.com",
        "api": "anthropic-messages"
      }
    }
  },
  "agents": {
    "defaults": {
      "memorySearch": {
        "provider": "openai",
        "model": "qwen3-embedding-4b",
        "remote": {
          "baseUrl": "https://api.agents.hypercli.com/v1"
        }
      }
    }
  }
}
```

## Shared Config File

Path:

```text
~/.hypercli/config
```

Example:

```bash
HYPERCLI_API_KEY=hyper_...
HYPERCLI_API_URL=https://api.hypercli.com
AGENTS_API_BASE_URL=https://api.hypercli.com/agents
AGENTS_WS_URL=wss://api.agents.hypercli.com/ws
```

Use the SDK helper to write it:

```python
from hypercli import configure

configure("hyper_...")
configure("hyper_...", "https://api.dev.hypercli.com")
configure(
    "hyper_...",
    "https://api.hypercli.com",
    "https://api.hypercli.com/agents",
    "wss://api.agents.hypercli.com/ws",
)
```

## Constructor Overrides

```python
from hypercli import HyperCLI

client = HyperCLI(
    api_key="hyper_...",
    api_url="https://api.hypercli.com",
    agent_api_key="sk-...",
    agent_dev=False,
    agents_api_base_url="https://api.hypercli.com/agents",
    agents_ws_url="wss://api.agents.hypercli.com/ws",
)
```

## Exported Constants

```python
from hypercli import GHCR_IMAGES, COMFYUI_IMAGE
```

- `GHCR_IMAGES`: base GitHub Container Registry namespace
- `COMFYUI_IMAGE`: default image used by `ComfyUIJob`
