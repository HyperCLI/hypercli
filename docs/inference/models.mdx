title: Models

---

Model discovery for inference should use HyperClaw endpoints.

## List Models

```bash
curl https://api.hypercli.com/v1/models \
  -H "Authorization: Bearer sk-..."
```

`/v1/models` is the discovery route. Scoped keys can allow model discovery with `models:*` even when actual inference is narrowed to one or more exact models.

## Use a Model

```bash
curl https://api.hypercli.com/v1/chat/completions \
  -H "Authorization: Bearer sk-..." \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kimi-k2.5",
    "messages": [{"role": "user", "content": "Say hello"}]
  }'
```

If you are minting narrowed keys, inference routes resolve the requested model from the request body. Typical selectors are:

- `model=kimi-k2-5`
- `model=glm-5`
- `model=qwen3-embedding-4b`

Aliases such as `kimi-k2.5-anthropic` still resolve back to the same canonical model for auth.
