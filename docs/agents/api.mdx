## Base URLs

| Surface | URL |
|---------|-----|
| Inference (OpenAI-compatible) | `https://api.hypercli.com/v1` |
| Agents API | `https://api.hypercli.com/agents` |
| Account + billing API | `https://api.hypercli.com/api` |
| Product UI | `https://agents.hypercli.com` |

<Note>
In production, routes are served under `/agents` (for example `/agents/stripe/{plan_id}` and `/agents/x402/{plan_id}`).
</Note>

Published spec: `https://api.hypercli.com/agents/openapi.json`

## Auth Modes

- Inference endpoints (`/v1/*`): `Authorization: Bearer sk-...`
- Account endpoints (`/api/*`): `Authorization: Bearer <jwt>`
- x402 checkout (`/agents/x402/{plan_id}`): JWT is optional

For scoped API keys:

- `GET /v1/models` uses the `models:*` baseline
- model inference routes resolve the requested `model` and can be narrowed with reserved scopes like `models:kimi-k2.5`
- subscription voice routes live under `/agents/voice/*` and use the `voice:*` family

## Plans

List plans:

```bash
curl https://api.hypercli.com/api/plans
```

Get current plan (JWT required):

```bash
curl https://api.hypercli.com/api/plans/current \
  -H "Authorization: Bearer <jwt>"
```

## Stripe Checkout (Credit Card)

Endpoint: `POST /agents/stripe/{plan_id}`

- Requires JWT auth
- Returns a Stripe-hosted `checkout_url`
- Webhook (`/api/stripe/hook`) activates subscription and applies team limits

```bash
curl -X POST https://api.hypercli.com/agents/stripe/basic \
  -H "Authorization: Bearer <jwt>" \
  -H "Content-Type: application/json" \
  -d '{
    "success_url": "https://agents.hypercli.com/plans?session_id={CHECKOUT_SESSION_ID}",
    "cancel_url": "https://agents.hypercli.com/plans?cancelled=true"
  }'
```

## x402 Checkout (USDC)

Endpoint: `POST /agents/x402/{plan_id}`

- Uses x402 HTTP 402 flow
- Middleware validates payment and accepts amounts `>=` plan minimum
- Route calculates duration from the signed transfer amount (`authorization.value`)

### Flow

1. POST request without payment headers
2. Receive HTTP 402 with payment requirements
3. Sign and retry with payment payload (`X-PAYMENT`)
4. Receive API key + expiry + limits

```bash
# Step 1
curl -X POST https://api.hypercli.com/agents/x402/basic

# Step 2: retry with x402 payment payload
curl -X POST https://api.hypercli.com/agents/x402/basic \
  -H "X-PAYMENT: <signed-payment-payload>"
```

### Duration Calculation

Duration scales with paid amount:

- `duration_days = (amount_paid / plan_price) * X402_PLAN_DURATION_DAYS`
- Includes sub-day precision (seconds)

So exact plan price gives base duration, and higher amounts increase duration proportionally.

### JWT Optional Behavior

- With JWT: creates account-linked subscription/payment records and updates team limits
- Without JWT: creates or reuses a wallet-backed user and returns a standalone API key for that purchase

### Response Shape

```json
{
  "ok": true,
  "key": "sk-...",
  "plan_id": "basic",
  "amount_paid": "35.000000",
  "duration_days": 30.0,
  "expires_at": "2026-03-15T12:00:00+00:00",
  "tpm_limit": 600000,
  "rpm_limit": 3000
}
```

## Balance-Funded Entitlements

Endpoint: `POST /agents/billing/balance/{plan_id}`

- Requires JWT auth
- Charges the linked Orchestra balance
- Creates a `grant`
- Applies that grant to create or extend the matching entitlement

Request body:

```json
{
  "duration": 2592000,
  "tags": ["customer=acme"]
}
```

Response shape:

```json
{
  "grant": {
    "id": "2b1c...",
    "type": "balance",
    "plan_id": "basic",
    "duration": 2592000,
    "tags": ["customer=acme"]
  },
  "entitlement": {
    "id": "6a54...",
    "starts_at": "2026-04-19T20:00:00+00:00",
    "expires_at": "2026-05-19T20:00:00+00:00",
    "plan_id": "basic",
    "tags": ["customer=acme"]
  },
  "payment": {
    "provider": "balance",
    "status": "succeeded"
  }
}
```

Notes:

- `duration` is in seconds
- if a matching entitlement already exists for the same `plan_id + tags`, the backend extends it instead of minting a parallel active window
- ownership is not encoded with hidden tags; entitlement binding remains a DB relationship

## Grant-Code Redemption

Endpoint: `POST /agents/billing/grants/redeem`

- Requires JWT auth
- Treats the code as an unapplied `grant`
- Applies that grant to create or extend the matching entitlement

Request body:

```json
{
  "code": "7f7d2f3a-1c4f-4d4d-8bdb-66a0df3d6f59"
}
```

Important:

- unactivated codes are `grants`, not `entitlements`
- redeemed codes become applied grants linked to an entitlement

## Job Tags

HyperClaw's x402 job creation endpoint accepts the same optional `tags` object used by Orchestra jobs:

```bash
curl -X POST https://api.hypercli.com/agents/x402/job \
  -H "Content-Type: application/json" \
  -H "X-PAYMENT: <signed-payment-payload>" \
  -d '{
    "gpu_type": "l40s",
    "gpu_count": 1,
    "docker_image": "nvidia/cuda:12.0",
    "command": "cHl0aG9uIHRyYWluLnB5",
    "tags": {
      "team": "ml",
      "env": "prod"
    }
  }'
```

When tags are set, they are returned on the nested `job` object in the response. The canonical published API reference for Agents is the OpenAPI spec at `https://api.hypercli.com/agents/openapi.json`.
