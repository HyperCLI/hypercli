HyperClaw exposes two chat surfaces:

- **Primary recommended:** Anthropic Messages API at `https://api.hypercli.com`
- **Compatibility option:** OpenAI-compatible API at `https://api.hypercli.com/v1`

For chat models, prefer the Anthropic Messages format. The OpenAI-compatible endpoint works well for broad tool compatibility, but tool calling quality is better with the Anthropic format.

`hyper config openclaw --apply` already configures the `hyperclaw` provider and the `/v1` memory-search remote automatically.

## Anthropic Messages API

Use this when your tool or SDK supports Anthropic-compatible providers:

```bash
curl https://api.hypercli.com/v1/messages \
  -H "x-api-key: sk-YOUR-API-KEY" \
  -H "anthropic-version: 2023-06-01" \
  -H "content-type: application/json" \
  -d '{
    "model": "kimi-k2.5-anthropic",
    "max_tokens": 512,
    "messages": [
      {
        "role": "user",
        "content": [
          { "type": "text", "text": "Hello!" }
        ]
      }
    ]
  }'
```

Use the OpenAI-compatible `/v1` endpoint below only when your integration requires it.

## OpenCode

Add HyperClaw as a provider in your `opencode.json` (or `.opencode.json`):

```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "hypercli": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "HyperCLI",
      "options": {
        "baseURL": "https://api.hypercli.com/v1",
        "apiKey": "sk-YOUR-API-KEY"
      },
      "models": {
        "kimi-k2.5": {
          "name": "kimi-k2.5"
        },
        "glm-5": {
          "name": "glm-5"
        }
      }
    }
  }
}
```

Then select the model with `hypercli/kimi-k2.5` or `hypercli/glm-5` in OpenCode.

## OpenClaw

Add a `hyperclaw` provider in `~/.openclaw/openclaw.json` under `models.providers`:

```json
{
  "models": {
    "mode": "merge",
    "providers": {
      "hyperclaw": {
        "baseUrl": "https://api.hypercli.com",
        "apiKey": "sk-YOUR-API-KEY",
        "api": "anthropic-messages",
        "authHeader": true,
        "models": [
          {
            "id": "kimi-k2.5",
            "name": "Kimi K2.5",
            "reasoning": true,
            "input": ["text", "image"],
            "contextWindow": 262144
          },
          {
            "id": "glm-5",
            "name": "GLM-5",
            "reasoning": true,
            "input": ["text", "image"],
            "contextWindow": 202752
          }
        ]
      }
    }
  },
  "agents": {
    "defaults": {
      "memorySearch": {
        "provider": "openai",
        "model": "qwen3-embedding-4b",
        "remote": {
          "baseUrl": "https://api.hypercli.com/v1",
          "apiKey": "sk-YOUR-API-KEY"
        }
      }
    }
  }
}
```

For chat, OpenClaw appends `/v1/messages` for `anthropic-messages`, so keep the chat `baseUrl` at `https://api.hypercli.com`, not `.../v1`.
For the branded HyperClaw setup, use `hyperclaw/kimi-k2.5` and `hyperclaw/glm-5` directly.

If you need compatibility with an older OpenAI-style OpenClaw config, this still works, but it is not recommended for chat:

```json
{
  "models": {
    "mode": "merge",
    "providers": {
      "hyperclaw": {
        "baseUrl": "https://api.hypercli.com/v1",
        "apiKey": "sk-YOUR-API-KEY",
        "api": "openai-completions",
        "models": [
          {
            "id": "kimi-k2.5",
            "name": "Kimi K2.5",
            "reasoning": true,
            "input": ["text", "image"],
            "contextWindow": 262144,
            "maxTokens": 8192
          }
        ]
      }
    }
  }
}
```

Then reference the model as `hyperclaw/kimi-k2.5` in your agent config, or set it as primary:

```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "hyperclaw/kimi-k2.5"
      },
      "models": {
        "hyperclaw/kimi-k2.5": {
          "alias": "kimi"
        }
      }
    }
  }
}
```

## Claude Code / Codex

Use environment variables — no config file needed:

```bash
export OPENAI_API_KEY="sk-YOUR-API-KEY"
export OPENAI_BASE_URL="https://api.hypercli.com/v1"

# Claude Code
claude --model hyperclaw/kimi-k2.5

# OpenAI Codex
codex --model hyperclaw/kimi-k2.5
```

## Python (OpenAI SDK)

```python
from openai import OpenAI

client = OpenAI(
    api_key="sk-YOUR-API-KEY",
    base_url="https://api.hypercli.com/v1"
)

response = client.chat.completions.create(
    model="kimi-k2.5",
    messages=[{"role": "user", "content": "Hello!"}]
)
print(response.choices[0].message.content)
```

## TypeScript (OpenAI SDK)

```typescript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: 'sk-YOUR-API-KEY',
  baseURL: 'https://api.hypercli.com/v1'
});

const response = await client.chat.completions.create({
  model: 'kimi-k2.5',
  messages: [{ role: 'user', content: 'Hello!' }]
});
```

## LiteLLM

Teams and organizations can request pooled inference if they want to put another
LiteLLM layer in front of HyperClaw for custom agents, internal tools, or AI
applications.

In that setup:

- your downstream LiteLLM does its own auth, aliases, and routing
- HyperClaw remains the upstream inference provider
- you use a normal HyperClaw API key as the upstream credential

You do not need to inject a custom authorization header beyond the normal
LiteLLM `api_key` field.

### OpenAI-compatible upstream

Use this when your downstream LiteLLM is routing through the OpenAI-compatible
`/v1` surface:

```yaml
model_list:
  - model_name: kimi-k2.6
    litellm_params:
      model: openai/kimi-k2.6
      api_base: https://api.hypercli.com/v1
      api_key: sk-YOUR-HYPERCLAW-KEY

  - model_name: glm-5
    litellm_params:
      model: openai/glm-5
      api_base: https://api.hypercli.com/v1
      api_key: sk-YOUR-HYPERCLAW-KEY
```

### Anthropic-style upstream

Use this when you want the recommended Anthropic Messages route for chat and tool
calling:

```yaml
model_list:
  - model_name: kimi-k2.6-anthropic
    litellm_params:
      model: anthropic/kimi-k2.6-anthropic
      api_base: https://api.hypercli.com
      api_key: sk-YOUR-HYPERCLAW-KEY

  - model_name: glm-5-anthropic
    litellm_params:
      model: anthropic/glm-5-anthropic
      api_base: https://api.hypercli.com
      api_key: sk-YOUR-HYPERCLAW-KEY
```

### Mixed downstream config

If you want to offer both surfaces from one downstream LiteLLM:

```yaml
model_list:
  - model_name: kimi-k2.6
    litellm_params:
      model: openai/kimi-k2.6
      api_base: https://api.hypercli.com/v1
      api_key: sk-YOUR-HYPERCLAW-KEY

  - model_name: kimi-k2.6-anthropic
    litellm_params:
      model: anthropic/kimi-k2.6-anthropic
      api_base: https://api.hypercli.com
      api_key: sk-YOUR-HYPERCLAW-KEY
```

Use the Anthropic-style entries for chat-first agent workloads. Keep the
OpenAI-compatible entries for SDKs, tools, and frameworks that expect `/v1`.

## Available Models

| Model | Context | Description |
|-------|---------|-------------|
| `kimi-k2.6-anthropic` | 262K tokens | Kimi K2.6 on the recommended Anthropic Messages route |
| `glm-5-anthropic` | 202K tokens | GLM-5 on the recommended Anthropic Messages route |
| `kimi-k2.6` | 262K tokens | Kimi K2.6 on the OpenAI-compatible route |
| `glm-5` | 202K tokens | GLM-5 on the OpenAI-compatible route |

Check available models at any time:

```bash
curl https://api.hypercli.com/v1/models \
  -H "Authorization: Bearer sk-YOUR-API-KEY"
```
