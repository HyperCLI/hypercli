HyperClaw exposes two chat surfaces:

- **Primary recommended:** Anthropic Messages API at `https://api.hypercli.com`
- **Compatibility option:** OpenAI-compatible API at `https://api.hypercli.com/v1`

For chat models, prefer the Anthropic Messages format. The OpenAI-compatible endpoint works well for broad tool compatibility, but tool calling quality is better with the Anthropic format.

`hyper agent config openclaw --apply` already configures both endpoints automatically.

## Anthropic Messages API

Use this when your tool or SDK supports Anthropic-compatible providers:

```bash
curl https://api.hypercli.com/v1/messages \
  -H "x-api-key: sk-YOUR-API-KEY" \
  -H "anthropic-version: 2023-06-01" \
  -H "content-type: application/json" \
  -d '{
    "model": "kimi-k2.5",
    "max_tokens": 512,
    "messages": [
      {
        "role": "user",
        "content": [
          { "type": "text", "text": "Hello!" }
        ]
      }
    ]
  }'
```

Use the OpenAI-compatible `/v1` endpoint below only when your integration requires it.

## OpenCode

Add HyperClaw as a provider in your `opencode.json` (or `.opencode.json`):

```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "hypercli": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "HyperCLI",
      "options": {
        "baseURL": "https://api.hypercli.com/v1",
        "apiKey": "sk-YOUR-API-KEY"
      },
      "models": {
        "kimi-k2.5": {
          "name": "kimi-k2.5"
        },
        "glm-5": {
          "name": "glm-5"
        }
      }
    }
  }
}
```

Then select the model with `hypercli/kimi-k2.5` or `hypercli/glm-5` in OpenCode.

## OpenClaw

Add a `hyperclaw` provider in `~/.openclaw/openclaw.json` under `models.providers`:

```json
{
  "models": {
    "mode": "merge",
    "providers": {
      "hyperclaw": {
        "baseUrl": "https://api.hypercli.com",
        "apiKey": "sk-YOUR-API-KEY",
        "api": "anthropic-messages",
        "models": [
          {
            "id": "kimi-k2.5",
            "name": "Kimi K2.5",
            "reasoning": true,
            "input": ["text", "image"],
            "contextWindow": 262144,
            "maxTokens": 8192
          },
          {
            "id": "glm-5",
            "name": "GLM-5",
            "reasoning": true,
            "input": ["text"],
            "contextWindow": 202752,
            "maxTokens": 8192
          }
        ]
      },
      "kimi-coding": {
        "baseUrl": "https://api.hypercli.com",
        "apiKey": "sk-YOUR-API-KEY",
        "api": "anthropic-messages",
        "headers": {
          "User-Agent": "claude-code/0.1.0"
        },
        "models": [
          {
            "id": "kimi-k2.5",
            "name": "Kimi K2.5",
            "reasoning": true,
            "input": ["text", "image"],
            "contextWindow": 262144,
            "maxTokens": 8192
          }
        ]
      },
      "hyperclaw-embed": {
        "baseUrl": "https://api.hypercli.com/v1",
        "apiKey": "sk-YOUR-API-KEY",
        "api": "openai-completions",
        "models": [
          {
            "id": "qwen3-embedding-4b",
            "name": "Qwen3 Embedding 4B",
            "reasoning": false,
            "input": ["text"],
            "contextWindow": 32768,
            "maxTokens": 8192,
            "mode": "embedding"
          }
        ]
      }
    }
  }
}
```

For chat, OpenClaw appends `/v1/messages` for `anthropic-messages`, so keep the chat `baseUrl` at `https://api.hypercli.com`, not `.../v1`.
For Kimi, the generated config also includes a `kimi-coding` provider alias so OpenClaw can apply its upstream Kimi-specific tool/thinking semantics while still using HyperClaw as the transport.

If you need compatibility with an older OpenAI-style OpenClaw config, this still works, but it is not recommended for chat:

```json
{
  "models": {
    "mode": "merge",
    "providers": {
      "hyperclaw": {
        "baseUrl": "https://api.hypercli.com/v1",
        "apiKey": "sk-YOUR-API-KEY",
        "api": "openai-completions",
        "models": [
          {
            "id": "kimi-k2.5",
            "name": "Kimi K2.5",
            "reasoning": true,
            "input": ["text", "image"],
            "contextWindow": 262144,
            "maxTokens": 8192
          }
        ]
      }
    }
  }
}
```

Then reference the model as `hyperclaw/kimi-k2.5` in your agent config, or set it as primary:

```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "hyperclaw/kimi-k2.5"
      },
      "models": {
        "hyperclaw/kimi-k2.5": {
          "alias": "kimi"
        }
      }
    }
  }
}
```

## Claude Code / Codex

Use environment variables — no config file needed:

```bash
export OPENAI_API_KEY="sk-YOUR-API-KEY"
export OPENAI_BASE_URL="https://api.hypercli.com/v1"

# Claude Code
claude --model kimi-k2.5

# OpenAI Codex
codex --model kimi-k2.5
```

## Python (OpenAI SDK)

```python
from openai import OpenAI

client = OpenAI(
    api_key="sk-YOUR-API-KEY",
    base_url="https://api.hypercli.com/v1"
)

response = client.chat.completions.create(
    model="kimi-k2.5",
    messages=[{"role": "user", "content": "Hello!"}]
)
print(response.choices[0].message.content)
```

## TypeScript (OpenAI SDK)

```typescript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: 'sk-YOUR-API-KEY',
  baseURL: 'https://api.hypercli.com/v1'
});

const response = await client.chat.completions.create({
  model: 'kimi-k2.5',
  messages: [{ role: 'user', content: 'Hello!' }]
});
```

## cURL

```bash
curl https://api.hypercli.com/v1/chat/completions \
  -H "Authorization: Bearer sk-YOUR-API-KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kimi-k2.5",
    "messages": [{"role": "user", "content": "Hello!"}],
    "stream": true
  }'
```

## Available Models

| Model | Context | Description |
|-------|---------|-------------|
| `kimi-k2.5` | 262K tokens | Kimi K2.5 — high-performance MoE model, vision + function calling |
| `glm-5` | 202K tokens | GLM-5 — 754B MIT-licensed reasoning model, record-low hallucination |

Check available models at any time:

```bash
curl https://api.hypercli.com/v1/models \
  -H "Authorization: Bearer sk-YOUR-API-KEY"
```
