## Overview

Job tags are optional raw tag strings you can attach at creation time:

```json
{
  "tags": ["team=ml", "env=prod"]
}
```

- Orchestra `POST https://api.hypercli.com/api/jobs` accepts `tags`
- Orchestra `GET https://api.hypercli.com/api/jobs` filters with repeated `tag=key=value`
- HyperClaw `POST https://api.hypercli.com/api/x402/job` also accepts `tags`
- Job responses include user tags plus system tags like `owner=<id>`, `resource=<id>`, and `job=<id>`

For the broader tag contract across agents, entitlements, jobs, and API keys, see [Tags](/agents/tags).

## Create A Tagged Job

```bash
curl -X POST https://api.hypercli.com/api/jobs \
  -H "Authorization: Bearer <api-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "gpu_type": "l40s",
    "gpu_count": 1,
    "docker_image": "nvidia/cuda:12.0",
    "command": "cHl0aG9uIHRyYWluLnB5",
    "tags": ["team=ml", "env=prod"]
  }'
```

Use the same `tags` array with HyperClaw x402 job creation at `POST https://api.hypercli.com/api/x402/job`.

## Filter Jobs By Tag

```bash
curl "https://api.hypercli.com/api/jobs?tag=team=ml&tag=env=prod" \
  -H "Authorization: Bearer <api-key>"
```

Each `tag` query param is an exact `key=value` match. In the CLI, the same filter is:

```bash
hyper jobs list --tag team=ml --tag env=prod
```
