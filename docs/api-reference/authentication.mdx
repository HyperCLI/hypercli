## API Key Authentication

Use your API key as a Bearer token:

```bash
curl https://api.hypercli.com/api/balance \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Or via `X-API-KEY` header:

```bash
curl https://api.hypercli.com/api/balance \
  -H "X-API-KEY: YOUR_API_KEY"
```

## Getting an API Key

1. Sign up at [hypercli.com](https://hypercli.com)
2. Go to [Console](https://console.hypercli.com)
3. Create an API key

## Managing Keys

Create, list, and delete keys via the API:

```bash
# Create key
curl -X POST https://api.hypercli.com/api/keys \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "my-key"}'

# List keys
curl https://api.hypercli.com/api/keys \
  -H "Authorization: Bearer YOUR_API_KEY"

# Delete key
curl -X DELETE https://api.hypercli.com/api/keys/{key_id} \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Inspect Current Auth Context

Use `GET /api/auth/me` to inspect the current auth principal.

For scoped API keys, this is the supported way to verify the key's own
capabilities and key identity metadata without requiring broader `user:*`
access.

## LLM API Authentication

The LLM API (`/v1/*` endpoints) uses the same API key but is OpenAI-compatible:

```python
from openai import OpenAI

client = OpenAI(
    api_key="YOUR_HYPERCLI_API_KEY",
    base_url="https://api.hypercli.com/v1"
)
```

For scoped keys:

- `GET /v1/models` uses the `models:*` baseline
- model inference routes can be narrowed with reserved scopes such as `models:kimi-k2.5`
- subscription voice routes use the `/agents/voice/*` surface with the `voice:*` family
