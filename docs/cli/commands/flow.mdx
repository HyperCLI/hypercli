## Overview

`hyper flow` is the recommended CLI surface for media generation.

It supports both:

- account billing (default)
- x402 pay-per-use (`--x402`)

When `--x402` is used, the CLI reads fixed flow pricing from `GET /flows`, creates an x402 payment with the embedded wallet, and calls `/api/x402/flow/{flow_type}`.

## Commands

```bash
hyper flow text-to-image <prompt>
hyper flow text-to-image-hidream <prompt>
hyper flow text-to-video <prompt>
hyper flow image-to-video <prompt> --image <url-or-path>
hyper flow speaking-video <prompt> --image <url-or-path> --audio <url-or-path>
hyper flow image-to-image <prompt> --image <url-or-path> [--image <url-or-path> ...]
hyper flow first-last-frame-video <prompt> --start <url-or-path> --end <url-or-path>
hyper flow text-to-speech <text>
hyper flow audio-to-text <audio-url-or-path>
```

## x402 Usage

```bash
# Uses fixed flow price from /flows
hyper flow text-to-image "a cyberpunk alley" --x402

# Explicit amount is optional, but must match fixed price exactly
hyper flow text-to-image "a cyberpunk alley" --x402 --amount 0.10
```

Successful `--x402` responses include:

- `render.id`
- `access_key` (`hypercli_access_...`)
- `status_url`
- `cancel_url`

Use that access key as Bearer token for polling/cancel calls (or from your agent runtime).

## Notes

- `hyper renders create --x402` is intentionally disabled.
- For pay-per-use, use `hyper flow ... --x402`.
- Local files are auto-uploaded when passed to image/audio options.
