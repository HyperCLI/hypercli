## Overview

`hyper agents` manages live HyperClaw deployments.

It covers:

- create / start / stop / delete
- wait and status
- file copy
- one-shot exec
- backend WebSocket shell
- backend log streaming
- gateway config, files, sessions, cron
- direct gateway chat

## Global Options

```bash
hyper agents --dev
hyper agents --agents-ws-url wss://api.agents.dev.hypercli.com/ws
```

## Current Route Defaults

OpenClaw-backed agents use:

- root host = gateway / agent
- `desktop-<host>` = desktop
- no default `shell-<host>` route

Extra routes are user-provided via the `routes` dict at create/start time.

## Lifecycle

### `create`

```bash
hyper agents create --name docs-demo --size medium --wait
```

Important options:

- `--name`
- `--size`
- `--no-start`
- `--wait / --no-wait`
- `--env`
- `--port`
- `--command`
- `--entrypoint`
- `--image`
- `--registry-url`
- `--registry-username`
- `--registry-password`
- `--gateway-token`
- `--dry-run`

### `wait`

```bash
hyper agents wait <agent_id> --timeout 300 --poll-interval 5
```

### `list`

```bash
hyper agents list
hyper agents list --json
```

### `status`

```bash
hyper agents status <agent_id>
```

### `start`

```bash
hyper agents start <agent_id>
```

### `stop`

```bash
hyper agents stop <agent_id>
```

### `delete`

```bash
hyper agents delete <agent_id> --force
```

## Exec, Copy, Shell, Logs

### `exec`

One-shot command through the backend exec API:

```bash
hyper agents exec <agent_id> "uname -a"
```

### `cp`

```bash
hyper agents cp ./local.txt <agent_id>:/workspace/local.txt
hyper agents cp <agent_id>:/workspace/output.log ./output.log
```

### `shell`

Interactive PTY through backend `/ws/shell/{agent_id}`:

```bash
hyper agents shell <agent_id>
```

This is the current shell path. It does not rely on a `shell-<host>` route.

### `logs`

```bash
hyper agents logs <agent_id> -n 200 -f
hyper agents logs <agent_id> --ws
```

Use `--ws` when you want the backend WebSocket path explicitly.

## Tokens And Env

### `token`

Refresh the current backend agent access token:

```bash
hyper agents token <agent_id>
```

### `budget`

Show current resource budget and usage:

```bash
hyper agents budget
```

## Gateway Commands

These commands target the OpenClaw gateway on the root agent host.

### `config`

```bash
hyper agents config <agent_id>
hyper agents config <agent_id> --schema
```

### `config-patch`

```bash
hyper agents config-patch <agent_id> '{"models":{"providers":{"openai":{"apiKey":"sk-..."}}}}'
```

### `models`

```bash
hyper agents models <agent_id>
```

### `files`

```bash
hyper agents files <agent_id>
hyper agents files <agent_id> --get SOUL.md
hyper agents files <agent_id> --set 'SOUL.md=# My Agent'
```

### `sessions`

```bash
hyper agents sessions <agent_id>
```

### `cron`

```bash
hyper agents cron <agent_id>
```

### `gateway-chat`

Current preferred chat path against the OpenClaw gateway:

```bash
hyper agents gateway-chat <agent_id> "Reply with exactly: hello"
hyper agents gateway-chat <agent_id> "Inspect the workspace" --session-key docs
```

This uses the root host gateway plus pairing-aware SDK logic.

### `chat`

This is older compatibility chat behavior and is not the preferred current OpenClaw path.
