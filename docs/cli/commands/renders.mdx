title: Renders And Flows
description: Direct renders are still supported alongside the higher-level flow surface.
---

HyperCLI supports two render surfaces:

- `hyper flow ...` for the managed flow catalog we curate
- direct renders via the REST API and SDKs when you want to work with raw render jobs yourself

`hyper flow` is the recommended CLI entrypoint for media generation today.

Use it when you want:

- stable managed templates
- flow-level pricing and x402 support
- higher-level commands like `text-to-image` and `image-to-video`

```bash
hyper flow text-to-image "a cat in space"
hyper flow text-to-video "a cat walking through snow"
hyper flow list
hyper flow get <render_id>
```

Use direct renders when you want lower-level control over render jobs via the API or SDKs:

- [Python renders SDK](/sdk/renders)
- [TypeScript renders SDK](/ts-sdk/renders)
- [Render API](/render)
