---
title: hyper renders
description: Lower-level managed render commands.
---

Use `hyper renders` when you want the older managed render API directly. For most new workflows, prefer `hyper flow`.

## list

```bash
hyper renders list
hyper renders list --state completed
hyper renders list --template hidream_i1_full
hyper renders list --type comfyui
hyper renders list --output json
```

## create

```bash
hyper renders create hidream_i1_full --prompt "a cat in space"
hyper renders create hidream_i1_full --prompt "a cat in space" --wait
hyper renders create hidream_i1_full --prompt "a cat in space" --gpu L40S --region kr
```

Key flags:

- `--prompt`, `-p`
- `--gpu`, `-g`
- `--region`, `-r`
- `--type`, `-t`
- `--wait`, `-w`
- `--notify`
- `--output`, `-o`

`--x402` is intentionally not supported here. Use `hyper flow ... --x402` instead.

## get

```bash
hyper renders get <render_id>
hyper renders get <render_id> --output json
```

## status

```bash
hyper renders status <render_id>
hyper renders status <render_id> --watch
hyper renders status <render_id> --output json
```

## cancel

```bash
hyper renders cancel <render_id>
```
