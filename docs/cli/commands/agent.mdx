`hyper agent` is the account and local-tooling surface for HyperCLI.

Use it for:

- onboarding
- wallet-based subscribe/login
- plan and model discovery
- provider config snippets
- local transcription
- voice and embedding helpers

## Commands

### `onboard`

Guided flow for:

- wallet creation
- funding
- plan selection
- subscription
- config generation
- verification

### `subscribe`

Subscribe to a HyperCLI plan through the x402 wallet flow.

```bash
hyper agent subscribe basic
hyper agent subscribe pro 100 --dev
```

### `status`

Show the saved HyperCLI key and current plan status.

### `plans`

List plan IDs, price, and limits.

```bash
hyper agent plans
hyper agent plans --dev
```

### `models`

List HyperCLI inference models.

```bash
hyper agent models
hyper agent models --json
```

### `login`

Wallet-based login that creates a user-bound HyperCLI API key and stores it locally.

### `config`

Generate config snippets for:

- `openclaw`
- `opencode`
- `env`

```bash
hyper config openclaw --apply
hyper config opencode --placeholder-env HYPER_AGENTS_API_KEY
hyper config env
```

Without `--apply`, the command prints the generated JSON blob to stdout.

OpenCode configs use `{env:NAME}` for environment variable interpolation:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "model": "hypercli/glm-5-anthropic",
  "provider": {
    "hypercli": {
      "npm": "@ai-sdk/anthropic",
      "name": "HyperCLI",
      "options": {
        "baseURL": "https://api.hypercli.com/v1",
        "apiKey": "{env:HYPER_AGENTS_API_KEY}"
      },
      "models": {
        "glm-5-anthropic": {
          "name": "glm-5-anthropic",
          "limit": { "context": 262144, "output": 65536 }
        },
        "kimi-k2.6-anthropic": {
          "name": "kimi-k2.6-anthropic",
          "limit": { "context": 262144, "output": 65536 }
        }
      }
    }
  }
}
```

The full generated `~/.openclaw/openclaw.json` example lives in:

- [Configuration](/cli/configuration)

With `--apply`, the command:

- writes `~/.openclaw/openclaw.json`
- runs `openclaw models list` to regenerate the active agent `models.json`

It does not restart OpenClaw automatically. Restart when you are ready:

```bash
openclaw gateway restart
```

### `openclaw-setup`

Patch `~/.openclaw/openclaw.json` directly using the saved HyperCLI key.

```bash
hyper agent openclaw-setup
hyper agent openclaw-setup --default
```

With `--default`, the generated OpenClaw config sets the primary coding model to
`hypercli/kimi-k2.5`.

### `exec` and `shell`

Compatibility forwards into `hyper agents`.

### `transcribe`

Local faster-whisper transcription, no remote API required.

### `voice`

Voice generation helpers.

### `embed`

Embedding helpers against HyperCLI.

## Files Written By `hyper agent`

- `~/.hypercli/agent-key.json`
- optionally `~/.openclaw/openclaw.json` when using `openclaw-setup`

## Recommended Flow

1. `hyper agent plans`
2. `hyper agent subscribe basic`
3. `hyper agent status`
4. `hyper config openclaw --apply`
5. `hyper agents create ...`
