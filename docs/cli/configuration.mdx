## Interactive Setup

```bash
hyper configure
```

This writes the shared config file at `~/.hypercli/config`.

## Environment Variables

| Variable | Description |
| --- | --- |
| `HYPERCLI_API_KEY` | Main HyperCLI API key |
| `HYPERCLI_API_URL` | REST API base URL |
| `HYPERCLAW_API_KEY` | HyperClaw key used by `hyper agents` and model discovery |
| `HYPERCLAW_API_BASE` | Override HyperClaw base URL for `hyper agent config` |
| `AGENTS_WS_URL` | Override backend WebSocket URL for `hyper agents` |
| `HYPERCLI_WALLET_PASSPHRASE` | Skip wallet passphrase prompts |

## Shared Config File

Location:

```text
~/.hypercli/config
```

Example:

```bash
HYPERCLI_API_KEY=hyper_...
HYPERCLI_API_URL=https://api.hypercli.com
```

## Local State Files

The CLI stores additional state in `~/.hypercli/`:

- `config`: shared API key and base URL
- `agent-key.json`: saved HyperClaw key from `hyper agent subscribe` or `hyper agent login`
- `agent-keys.yaml`: key history written by `hyper agent subscribe`
- `agent-jwt.json`: wallet-auth JWT used during agent login flows
- `agents.json`: cached agent metadata for ID prefix resolution
- `wallet.json`: encrypted wallet keystore
- `x402_renders.jsonl`: saved x402 render history used by `hyper flow renders`

## Priority Rules

The main CLI resolves configuration in this order:

1. Explicit flags
2. Environment variables
3. `~/.hypercli/config`

The `hyper agents` command group resolves agent auth in this order:

1. `HYPERCLAW_API_KEY`
2. `~/.hypercli/agent-key.json`
