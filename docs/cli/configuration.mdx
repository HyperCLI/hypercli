## Interactive Setup

```bash
hyper configure
```

This writes the shared config file at `~/.hypercli/config`.

## Environment Variables

| Variable | Description |
| --- | --- |
| `HYPER_API_KEY` | Main HyperCLI API key |
| `HYPER_API_BASE` | Primary HyperCLI REST API base URL |
| `HYPER_AGENTS_API_KEY` | Agent-scoped API key used by `hyper agents` |
| `HYPERCLI_WS_URL` | Primary HyperCLI WebSocket base URL |
| `AGENTS_API_BASE_URL` | HyperClaw deployments API base URL |
| `AGENTS_WS_URL` | HyperClaw backend WebSocket base for logs and shell |
| `HYPERCLI_WALLET_PASSPHRASE` | Skip wallet passphrase prompts |

## URL Model

The CLI now separates the primary product API from the agents API:

- `HYPER_API_BASE`
  - billing
  - jobs
  - keys
  - account APIs
- `HYPERCLI_WS_URL`
  - primary WebSocket base derived from the main API
- `AGENTS_API_BASE_URL`
  - HyperClaw deployments REST base
  - create/start/stop/delete
  - files
  - exec
  - desktop token APIs
- `AGENTS_WS_URL`
  - HyperClaw backend WebSocket base
  - `/logs/{agent_id}`
  - `/shell/{agent_id}`

Recommended production values:

```bash
HYPER_API_BASE=https://api.hypercli.com
HYPERCLI_WS_URL=wss://api.hypercli.com
AGENTS_API_BASE_URL=https://api.hypercli.com/agents
AGENTS_WS_URL=wss://api.agents.hypercli.com/ws
```

Recommended dev values:

```bash
HYPER_API_BASE=https://api.dev.hypercli.com
HYPERCLI_WS_URL=wss://api.dev.hypercli.com
AGENTS_API_BASE_URL=https://api.dev.hypercli.com/agents
AGENTS_WS_URL=wss://api.agents.dev.hypercli.com/ws
```

### OpenClaw Base URL Rule

`hyper config openclaw` uses the agents inference host derived from `AGENTS_API_BASE_URL`.

Important:

- `hyperclaw` and other `anthropic-messages` providers should use the bare host, for example:
  - `https://api.agents.hypercli.com`
- do not append `/v1`
- OpenClaw appends `/v1/messages` itself
- embeddings and other OpenAI-compatible providers should use `/v1`

### OpenClaw Sync Rule

`hyper config openclaw --apply` does two things:

1. writes `~/.openclaw/openclaw.json`
2. runs `openclaw models list` to regenerate `~/.openclaw/agents/main/agent/models.json`

It does **not** restart the gateway for you.

If OpenClaw is already running, restart it yourself when ready:

```bash
openclaw gateway restart
```

If local OpenClaw state drifts, the safe recovery flow is:

```bash
rm -f ~/.openclaw/agents/main/agent/models.json
rm -f ~/.openclaw/agents/main/agent/auth-profiles.json
hyper config openclaw --apply
openclaw gateway restart
```

### Example OpenClaw Config

This is the shape generated by:

```bash
hyper config openclaw
```

Sanitized example with the HyperClaw-specific bits called out:

```json
{
  "models": {
    "mode": "merge",
    "providers": {
      "hyperclaw": {
        "baseUrl": "https://api.agents.hypercli.com",
        "apiKey": "hyper_api_I0Gr****Obui",
        "api": "anthropic-messages",
        "authHeader": true,
        "models": [
          {
            "id": "kimi-k2.5",
            "name": "Kimi K2.5",
            "reasoning": true,
            "input": ["text", "image"],
            "contextWindow": 262144
          },
          {
            "id": "kimi-k2.5-anthropic",
            "name": "Kimi K2.5 Anthropic",
            "reasoning": false,
            "input": ["text", "image"],
            "contextWindow": 200000
          },
          {
            "id": "glm-5",
            "name": "GLM-5",
            "reasoning": true,
            "input": ["text", "image"],
            "contextWindow": 202752
          }
        ]
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "hyperclaw/kimi-k2.5"
      },
      "models": {
        "hyperclaw/kimi-k2.5": { "alias": "kimi" },
        "hyperclaw/kimi-k2.5-anthropic": { "alias": "kimi" },
        "hyperclaw/glm-5": { "alias": "glm" }
      },
      "memorySearch": {
        "provider": "openai",
        "model": "qwen3-embedding-4b",
        "remote": {
          "baseUrl": "https://api.agents.hypercli.com/v1",
          "apiKey": "hyper_api_I0Gr****Obui"
        }
      }
    }
  }
}
```

Notes:

- `models.providers.hyperclaw.baseUrl` is the bare agents inference host
- `api: "anthropic-messages"` is required for OpenClaw chat/message routing
- `authHeader: true` tells OpenClaw to send `Authorization: Bearer ...`
- `agents.defaults.model.primary` should usually be `hyperclaw/kimi-k2.5`
- `memorySearch.remote.baseUrl` does use `/v1`
- the example key is intentionally obfuscated

If you do not want to write a literal key into `openclaw.json`, generate placeholders instead:

```bash
hyper config openclaw --placeholder-env HYPER_API_KEY
```

That produces:

```json
{
  "models": {
    "providers": {
      "hyperclaw": {
        "apiKey": "${HYPER_API_KEY}"
      }
    }
  },
  "agents": {
    "defaults": {
      "memorySearch": {
        "remote": {
          "apiKey": "${HYPER_API_KEY}"
        }
      }
    }
  }
}
```

## Shared Config File

Location:

```text
~/.hypercli/config
```

Example:

```bash
HYPER_API_KEY=hyper_...
HYPER_API_BASE=https://api.hypercli.com
AGENTS_API_BASE_URL=https://api.hypercli.com/agents
AGENTS_WS_URL=wss://api.agents.hypercli.com/ws
```

## Local State Files

The CLI stores additional state in `~/.hypercli/`:

- `config`: shared API key and base URL
- `agent-key.json`: saved HyperClaw key from `hyper agent subscribe` or `hyper agent login`
- `agent-keys.yaml`: key history written by `hyper agent subscribe`
- `agent-jwt.json`: wallet-auth JWT used during agent login flows
- `agents.json`: cached agent metadata for ID prefix resolution
- `wallet.json`: local wallet file, either an encrypted keystore or a plaintext private-key wallet

## Priority Rules

The main CLI resolves configuration in this order:

1. Explicit flags
2. Environment variables
3. `~/.hypercli/config`

The `hyper agents` command group resolves agent auth in this order:

1. `HYPER_AGENTS_API_KEY`
2. `HYPER_API_KEY`
3. `~/.hypercli/agent-key.json`
